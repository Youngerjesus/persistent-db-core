  - Contract line 55 lists expected visible rows `[(1, 'seed'), (2, 'committed_before_tail')]`, while the plan/design still leave room for a documented storage-error path.

  - Current WAL uses a frame `state` byte rather than a separate post-frame commit marker.
  - The plan should state the exact non-committed fixture form to use, such as a complete rolled-back frame or another current-format non-committed state that is accepted by replay and skipped, and require the final report to document this mapping.
