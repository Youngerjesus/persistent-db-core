## Provenance Contract
- Evidence-heavy: yes. Acceptance depends on generated benchmark evidence, command sentinels, current-run provenance, and an artifact-producing harness.
- Evidence root: `target/bench_acceptance/`.
- Required artifacts:
  - `target/bench_acceptance/section14-benchmark-acceptance.json`
  - `DB_BENCH: PASS evidence=target/bench_acceptance/section14-benchmark-acceptance.json`
  - `BENCH_ACCEPTANCE: PASS evidence=target/bench_acceptance/section14-benchmark-acceptance.json`
  - Updated docs sections in `docs/cli_contract.md`, `docs/benchmarks.md`, `docs/performance_report.md`, `docs/v1_acceptance.md`, `docs/bug_diary.md`
  - Scheduler result at the task run's `result.md`
- Evidence IDs / scenario IDs:
  - `METRIC-14-1` maps to `S14-S3`, `S14-S4`, `S14-S6`
  - `METRIC-14-2` maps to `S14-S1`, `S14-S5`
  - `METRIC-14-3` maps to `S14-S5`, `S14-S6`, `S14-S13`
  - `METRIC-14-4` maps to `S14-S8`, `S14-S13`
  - `FAIL-14-5` maps to `S14-S7`
  - `EVID-15` maps to `S14-S10`, `S14-S11`, `S14-S14`, `S14-S15`
  - `EVID-16-7` maps to `S14-S11`
- Current-run ID source: task metadata `active_run_id=qa_prep_retry_1_resume_20260519_161703_348075_43689e72`; implementation evidence must be regenerated by commands executed in the current implementation or verification pass, not copied from older `target/bench_acceptance/*` files.
- Clean generation rule: canonical launch evidence for a fresh repair or verification pass is deleted, replaced, or regenerated from the current run. Historical artifacts may remain only as audit evidence and must not be reused as current proof.
- No artifact reuse rule: `target/bench_acceptance/v1-bench-docs-acceptance.json` and any older 1k benchmark outputs are invalid as Section 14 proof, even if present.
- Writer/validator separation expectation: `db bench` writes truthful pre-verifier evidence with `commands.verify_bench_acceptance.status="pending"`; `scripts/verify_bench_acceptance` invokes public `db bench`, validates schema/thresholds/sentinels, then atomically finalizes `commands.verify_bench_acceptance.status="pass"` and `result="pass"` in the same evidence file.
- Redaction target list: no secrets or user data are expected. Evidence must not include machine-specific temp paths outside repo-relative artifact paths, environment secrets, full shell environments, or uncontrolled wall-clock/random seeds.
